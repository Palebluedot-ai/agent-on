## 二、默认工作目录 B（产品钉死，跨 OS）

| OS | 默认 `work_root`（可写 clone） |
|---|---|
| **macOS / Linux** | `~/.local/share/agent-on` |
| **Windows** | `%LOCALAPPDATA%\agent-on`（通常 `C:\Users\<你>\AppData\Local\agent-on`） |

| 配置 | 路径 |
|---|---|
| 本机配置 | `~/.config/agent-on/config.json` → `{"work_root":"<上表路径>"}` |
| 覆盖 | `AGENT_ON_ROOT` 环境变量优先于配置与默认目录 |

**不再**以 `~/Projects/Agent-On` 为产品默认（Chao 本机历史路径可继续用 config 指过去，但不进协议）。

解析序（`agent_on_paths.py`）：

1. `AGENT_ON_ROOT`
2. `~/.config/agent-on/config.json` 的 `work_root`
3. 上溯项目 `agent-on.lock.md`「本地路径」
4. **默认目录**（上表）若已是合法 agent-on 仓

## 三、setup 脚本职责

`scripts/setup.py`（跨平台，标准库 + `git`/`python3`）：

1. 解析默认或 `--work-root` 目录  
2. 无仓则 `git clone` 官方 HTTPS 源；有仓则 `fetch`  
3. checkout **`--pin`**（默认 `v0.5.0`）  
4. 写 `~/.config/agent-on/config.json`  
5. 可选：尝试 `claude`/`codex` plugin marketplace add + install（失败只警告）  
6. 可选：symlink skill 到 `~/.claude/skills` / `~/.agents/skills`  
7. 跑 doctor + 对本仓 `intake/` 跑 `ledger/intake-lint.py`（无卡则跳过）  
8. 打印三家开工一句话  

**不**做：改 shell RC 不经确认、force-push、删用户已有 work_root 内容（已有仓只 fetch/checkout）。

## 四、intake-lint

权威脚本：`ledger/intake-lint.py`（Promotion Card 六项 + evidence 硬门）。  
setup 收尾调用；贡献 PR 建议 CI 或本地强制跑。消化开场可选用。
