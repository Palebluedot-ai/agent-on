# 阶段闸门:申请切阶段 ≠ 切阶段

> 职责边界:本篇管「什么算继续当前阶段、什么算申请切阶段、什么必须拦」,以及冻结物怎么改;不管某个阶段内部具体怎么干。
> 源流:二代 communication-governance-playbook/docs/conversation-state-machine-v1.md + 01_protocol/boundary-confirmation-v1.md,2026-07-07 批二移植

## 一、漂移的机制:两种输入最容易被误当成切阶段

阶段漂移(drift)不是模型「乱来」,是**语义混淆**——两类完全正常的输入,长得太像「换阶段」:

1. **用户补充背景 / 追问细节 / 要求重印当前文档 / 让你解释当前主线**——这些是在**深化当前阶段**,不是要换。
2. **用户临时冒个想法,或 AI 自己建议「下一步可以做 X」**——这是**提议**,不是**批准**。

把这两类当成切阶段,后果是主线被澄清对话带偏:你以为在推进,其实已经离开了用户锁定的那一步。所以第一条原则就是——**澄清与灵感,默认不触发切阶段。**

## 二、三态语义:提议 / 申请 / 执行必须拆开

二代把它收成一个状态机,核心是三个动作分层。搬到 Claude Code 语境:**orchestrator 主会话就是闸门持有者**——只有它能改 `current_phase`,子代理只干活、只汇报,碰不到闸门。

| 动作 | 含义 | 谁能做 | 会不会切阶段 |
|---|---|---|---|
| `continue_phase` | 继续当前阶段(补背景、追问、重印、加 note、更新 to-do) | 任何输入默认落这里 | **不会** |
| `request_transition` | 申请切阶段:当前阶段文档已存在且被明确「请审阅 / 请通过」,用户明确表示要进下一阶段,系统能指出目标阶段 | 用户提出 | **不会**(只是进入「是否批准」的判断点) |
| `perform_transition` | 真正切:前置条件全满足才放行 | orchestrator 主会话 | **会** |

`perform_transition` 的前置条件(少一个都不放行):① 当前阶段文档存在;② 达到最低契约(验收 ≤8 条那种);③ 被**显式批准**;④ 目标阶段是当前阶段的合法下一步(不许跳级)。

**记住这句:`request_transition` 不是切阶段本身,它只表示「当前阶段到了该不该批准的判断点」。** 申请 ≠ 生效。

## 三、6 个 block 理由码(保留原码名)

拦截时必须带一个理由码,让「为什么不放行」可审计、可复现:

| 理由码 | 触发场景 |
|---|---|
| `BLOCK_MISSING_DOC` | 必需文档缺失,却想继续推进 |
| `BLOCK_DOC_NOT_APPROVED` | 当前阶段未批准,却想进下一阶段 |
| `BLOCK_ILLEGAL_TRANSITION` | 想跳级 / 走非法的阶段跳转 |
| `BLOCK_NEXT_MUTATION` | 未经显式批准,想改写 `Next`(下一步) |
| `BLOCK_SCOPE_VIOLATION` | 动作越出当前阶段的 In-Scope |
| `BLOCK_SOURCE_OF_TRUTH_CONFLICT` | 承接 / 举证材料与当前 Canonical 冲突,又没先收口(见 [truth-hierarchy.md](truth-hierarchy.md)) |

## 四、冻结物要改,必须显式两步(fail-closed)

有几样东西是**冻结物**:阶段契约、暂停项(Out-of-Scope 禁令)、锁定的 `Next`。它们的默认状态是**锁**——出现任何模糊,系统偏向 **fail-closed**(拦下,别放行)。

改冻结物,走双层解锁,两步都要,缺一不解:

1. **用户口令**:用户明确说要切 / 要改(比如「切阶段到 …」「暂停当前,改做 …」「重排优先级」)。
2. **指名对象**:说清改的是哪一个——目标阶段是哪个、要动的 `Next` 是哪一条。

只有口令没指名,或只指名没口令,都不解锁,保持冻结。这样澄清对话再多,也带不动冻结物。

## 五、历史教训:别照搬重防御

二代最早的实现是**每轮口令复述硬拦**——每条回复末尾都得贴 Global / In-Scope / Out-of-Scope / Next / Drift Check 一整套,靠固定中文口令逐轮闸门。那是弱模型时代的产物:模型记不住主线,只能靠机械复述硬顶。

**强模型时代不要照搬。** 现在的做法是「**前置对齐 + 闸门只在切阶段时启用**」:

- 主线、In-Scope、暂停项写进 `AGENTS.md` 和 `docs/state/progress.yaml`——文件持久,不靠逐轮复述(prompt 易挥发,文件系统持久)。
- 闸门平时不响,**只在真要 `perform_transition` 时才启用**那四个前置条件检查。
- 会话续接时做一次握手对齐(见 [../boot/session-handshake.md](../boot/session-handshake.md)),而不是每轮都背一遍。

保留的是**精神**(三态分离、Next 不被澄清带偏、fail-closed);扔掉的是**形式**(每轮口令复述)。把重防御当默认,只会淹没真正该拦的那一次。
