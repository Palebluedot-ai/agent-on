# 沉淀分层:记了一堆 loop-notes,到底该归哪

> 职责边界:本篇管「一条沉淀属于哪一层、该落到 agent-on 五块的哪一块」,不管某条沉淀的正文怎么写。解决的是实际痛点——热的时候记了一堆东西,冷下来发现不知道归档到哪、下次也捞不出来。
> 源流:支线 evolving-memory-system/04_protocols/agent-orchestration-classification-v1.md §Soul Memory,2026-07-07 批二移植

## 一、五层沉淀(L0→L4)

「沉淀」不是一种东西。把它全写成同一种(比如全塞进一个 notes 文件),结果就是又长又捞不出来。按**可复用范围**和**生命周期**分五层:

| 层 | 是什么 | 一句人话 | 例子 |
|---|---|---|---|
| **L0 Operational** | 运行态临时数据 | 这次跑了啥,过程数据 | run_id、墙钟、冲突数、重试计数 |
| **L1 Episodic** | 单次任务复盘 | 这一次发生了什么、结论是啥 | 「Run #3 撞车:冻结 fixture 时顺序没写死」 |
| **L2 Semantic** | 可复用知识 | 跨任务都成立的事实/教训 | 「本地 vercel dev 行为 ≠ 生产,兜底写服务端」 |
| **L3 Procedural** | 可复用流程/协议 | 下次照着做的规则 | 「合流 checklist 固定一条:grep 全部 pumpWidget 钉全量 Fake」 |
| **L4 Identity** | 用户稳定偏好 | 这个人一贯要什么 | 「Python 一律 uv;文案中英不混杂」 |

生命周期从上到下递增:L0 短期(用完可弃)、L1 中期(可追溯)、L2/L3 长期(可版本迭代)、L4 长期(基本不变)。

## 二、升层判据(别什么都往上塞)

分层的价值全在「什么该升、什么不该升」。乱升层比不分层还糟——L2/L3 一旦塞满一次性的东西,就没人信、没人查了。三条硬判据:

1. **只有「跨任务可复用」才升 L2/L3**。这次任务专属的现象(某个具体 bug 的具体修法)留在 L1;只有当它抽象成「这类情况都这么办」时,才升 L2(知识)或 L3(流程)。判断法:问一句「下个项目/下个切片还用得上吗?」用不上就压在 L1。
2. **高频失败模式优先沉淀为 L3 anti-pattern**。反复踩的坑,价值最高的沉淀形态不是「记一笔教训」(L1),而是「固化成一条防它再犯的规则」(L3)。比如 fork 撞车踩过就写进协议「单会话单写者」,而不是每次都复盘一遍。
3. **只有用户稳定偏好进 L4**。一次性的口味(「这个按钮我想要蓝的」)不是 L4;一贯的原则(「界面同语言版不许中英混杂」)才是。L4 少而稳,进去的东西要能管一整个项目。

## 三、映射到 agent-on 五块(该沉到哪)

分层不是为了分类学好看,是为了**每条沉淀有唯一的家**。对上 agent-on 的五块资产:

| 沉淀层 | 落到哪一块 | 物理位置 |
|---|---|---|
| L0 运行数据 | **Ledger** | `ledger/runs/<run_id>.jsonl`(机器台账)+ run-ledger 主表 |
| L1 单次复盘 | **Ledger / Bench** | run-log 明细段;成模式的翻车进 `bench/loop-notes.md` |
| L2 可复用教训 | **Bench 案例集** | `bench/loop-notes.md` 的「可复用 loop」条目 |
| L3 流程规则 | **Kit / Playbook** | 规则原文进 playbook,固化成模板/checklist 进 kit |
| L4 用户偏好 | **项目 AGENTS.md** | 硬约束进 AGENTS.md(跨工具宪法),不进临时笔记 |

两个易错点:①**L1 和 L2 在同一个文件里也要分清**——`loop-notes.md` 里「某次现象+根因+修法」是 L1,末尾抽出的「可复用 loop / 通用化条件」才是 L2,写的时候就该把后者单独拎一段,不然下次捞不到。②**L3 别停在 Playbook**——一条流程规则只写进 playbook 是「知道了」,真正生效要落进 kit 的模板或 checklist(比如 merge-checklist.md 加一行),让它下次自动被执行,而不是靠人记得。

一句话带走:**沉淀的唯一去处由「可复用范围」决定——只这次用的压 L1 进 Ledger,反复用的升 L3 进 Kit;分层不是为了归档整齐,是为了下次真能捞出来、真能自动生效。**
