# ABDC 多轨决策:选型不收敛且返工贵时,怎么选

> 职责边界:本篇管「多个方案里到底选哪个」的决策纪律,不管某个方案内部怎么实现。sop.md 管流程(怎么从需求走到上机),ABDC 管流程里那个「选哪条路」的岔口。
> 源流:支线 evolving-memory-system/planning/abdc/(全套论证实例)+ templates/(四个模板),2026-07-07 批二移植

## 一、什么时候动用 ABDC

sop.md 的 Phase 2 是**多声音评审**,那是给「已经大致定了方向的计划」挑毛病。但有一类岔口评审兜不住:**方案本身不收敛,而且选错了返工很贵**——技术栈选型、架构分叉、产品方向。这种岔口的特征是:好几个候选都说得通、拍脑袋定了半个月后才发现走错、推翻要重写一大片。

ABDC 就是给这种岔口的。它和评审环是**上下游**关系:ABDC 先定「走哪条路」,定完了这条路的计划再进 sop.md 的评审环挑毛病。别把两者混用——评审环不负责在几个大方向里做选择,ABDC 也不负责抠某个方案的实现细节。

判据一句话:**候选 ≥ 2 个都不明显劣、且反悔成本高于一次论证的成本,就上 ABDC;否则直接进评审环。**

## 二、四步法(A→B→C→D)

名字来自四份产物,顺序不能乱:

**A. Decision Brief — 框定问题,列 2~4 个选项。**
一页纸讲清:要解决什么、为什么现在必须决、不决的代价;然后列 2~4 个选项,每个写核心思路 / 预期收益 / 主要风险 / **可回滚性(high/medium/low)**。再定本次的评估标准和权重(总和 100)。选项超过 4 个说明问题没框清,回去收敛。

**B. 三条论证轨并行(A/B/C 各一份 Track Memo)。**
把选项交给三条独立的轨道去论证,惯例是保守轨 / 平衡轨 / 激进轨。每轨自己写核心假设、机制设计、预期收益。**精髓在「反例义务」:每轨必须自己写出至少一条能打倒自己的强反例 + 一条缓解策略**。没有反例的 memo 不算数——这条逼着论证者跳出「我的方案最好」的自证陷阱。

**C. Review Packet — 加权评审,强制记录异议。**
一个独立评审席位,按 A 定的标尺和权重给三轨逐项打分、算加权总分,选出 Top1/Top2、淘汰项。**这一步的精髓是「异议记录栏」——凡有分歧,必须写清是什么异议、怎么处理的、哪些仍未解决(open)**。异议不记,收敛就是假的:分数堆出来一个 Top1,但没人知道当初的反对意见去哪了,过两周又原地争一遍。开放异议要挂到下一轮明确处理。

**D. Experiment Card — 只给 Top1 做最小可证伪实验。**
不是选出 Top1 就开工,而是先花很小的成本证伪它。一张卡只验**一个**假设(如果 X 那么 Y 因为 Z),动作要最小、**≤ 5 个工作日**、成本上限写死。判定三态:**Pass / Fail / Inconclusive**——Inconclusive 时怎么办要提前写(比如「只允许延 2 天补证,不许新增轨道」)。必须带**熔断条件 + 回滚步骤 + 最大可接受损失**。跑完按三态各自的决策出口走。

## 三、在 Claude Code / Codex 里怎么落地

四步天然映射到 orchestrator + 子代理:

- **A(Brief)**:主会话(orchestrator)写,用户拍板评估标准和权重——这是人类闸门,不下放给子代理。
- **B(三轨)**:开**三个并行子代理**,各写一份 Track Memo(各自独立 context,互不看对方的稿,才有独立性)。物理上可以给每轨一个 worktree,产物落到 `docs/decisions/<议题>/track-memo-{A,B,C}.md`。
- **C(评审)**:另起一个**独立审查子代理**(带对抗心态,不是刚写完 memo 的那个),吃三份 memo 产出 Review Packet。独立性是关键——自己评自己的方案等于没评。
- **D(实验)**:主会话据 Top1 写 Experiment Card,实验本身按普通 phase 卡执行(TDD、验证后才说完成那一套照旧)。

模板在 `../kit/abdc/`(四个),直接抄字段。整个论证过程的产物都归档进 Canonical 层的 `docs/decisions/`,不是散在聊天记录里——决策要能被后人复核,不然半年后又是一笔糊涂账。

一句话带走:**评审环挑毛病,ABDC 做选择;选择的可信度不在最后那个分数,而在每轨的反例和评审里被记下来的异议。**
