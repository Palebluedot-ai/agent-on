# AI 脚手架防幻觉设计 — 12 条机制(Euan-Flutter 实证)

> 核心认知:**幻觉不是模型缺陷,是信息缺口的必然产物**。模型在缺口上会用「最像真的」内容补位。
> 脚手架的职责 = 把每个缺口要么填上机械事实,要么钉上显式禁令,让「编造」无处落脚。
> 每条机制都标了本项目的实证(✓=正面证据,✗=缺了这条时的翻车)。

## A 类:堵「接口幻觉」(两个 AI 各自想象对方)

1. **契约 fixtures 双端咬合**:前后端消费**同一份 JSON 文件**做断言,接口形状无法「各自记忆」。
   ✓ Run #2:前端怀疑 PATCH 响应形状,合流验证零改动闭合。
   ✗ Run #3:fixture 的**排序语义**没冻结(只锁了形状),两轨各信一边,合流撞车——教训:**冻结要连语义一起冻**(排序/上限/空值/口径)。
2. **悬点显式化**:轨道报告强制带「契约悬点」栏——AI 必须把假设交出来,而不是埋进代码。
   ✓ Run #4:13 个悬点合流对账全相容,零返工;悬点栏让 display_name 可空性这种地雷在合流前就摆上桌面。
3. **参考模式指针**:「照 customers-repo.ts 的 cursor 写法」替代三段文字描述。既有代码是不会说谎的规格。

## B 类:堵「状态幻觉」(AI 记错项目进行到哪了)

4. **单一状态写者**:progress.yaml 只有 orchestrator 写;轨道 agent 不写状态、不 push。多写者=状态漂移=后续所有推理建立在错误前提上。
5. **自包含 phase 卡**:setpoint + 验收 + 禁区 + 指针全部卡内解决,`self_contained: true` 是承诺——执行者**不需要回忆**会话历史。上下文断了(compaction/断连)也能原地续跑。**长 phase / 跨会话多 Loop**:在 phase 卡内维护 **append-only Loop 台账**(日期·目标·验收命令·结果),新会话读 progress.next + 台账最后一行续跑——模板见 kit/phase-card-template §2b(IPONews 2026-07-16 实证)。
   ✓ 本项目跨 compaction、跨 API 断连多次,零迷路。
6. **disturbance = 显式负空间**:「不许碰 app/」「不许发明业务规则」「通知=纯客户端,别在后端做」。AI 的幻觉经常以「顺手多做」的形态出现,禁区条款与验收标准同权重。

## C 类:堵「完成幻觉」(声称做了但没做/没验)

7. **TDD 铁律**:没有失败测试不许写生产代码。测试先红后绿 = 行为规格先于实现存在,「我觉得逻辑对了」不再是过关条件。
8. **验证后才说完成**:任何「完成」声明必须附**验证命令的实际输出**(测试计数行/curl 响应),禁止「应该没问题」。
   ✓ 每轨报告都带输出末行;S5b 报告里 Stripe LIVE 标 ⏸ 挂账而非谎称走通。
9. **独立对抗审查**:切片收口由另一个带找茬心态的 agent 复核,failed→respond→passed 留痕。第一个 AI 的 checkbox 自评不可信——它有「让自己通过」的结构性动机。
   ✓ S2 审查抓到 T17 重写时被误删的并发双确认测试(Critical 级),主执行者自己完全没意识到。
10. **挂账文化**:外部依赖缺位(Stripe key/DNS)时,验收项标 ⏸ + 写清事后验证步骤。给 AI 一条「诚实但不算失败」的出路,它就不需要伪造。**prompt 里预先写明「严禁伪造证据」比事后抓假更便宜。**

## D 类:堵「决策幻觉」(AI 替人做产品决定)

11. **暂停项写成禁令**:用户说「以后再聊」的(加密策略/状态机),不是从计划里删掉,而是写进 AGENTS.md:「未获明确指令前不实现、不假设」。删掉=留缺口=AI 某天自作主张补上;禁令=缺口被钉死。
    ✓ 用户 7-04 问「加密进计划了吗」,一查禁令条款即答,零歧义。
12. **高幻觉区纯函数化 + 边界钉死**:时区换算、ISO 周、金额、幂等——AI 最容易「差不多先生」的地方,一律抽纯函数 + 边界单测(HK 23:59/UTC 跨日/2026-W53/双投递收敛)。安全属性(防枚举「响应逐字节相同」)写成**测试断言**而不是「注意安全」。
    ✓ B#3:模型拒绝为「下周一」编日期(没参考日期时 due_at 留 null)——好的信任边界让模型的诚实可被利用:prompt 注入今天日期后问题根治,而不是逼模型猜。

## 附:第五类——偏好缺口(2026-07-05 官网 v1 返工后增补)

前四类都在事实域(有客观对错)。第五类在**偏好域**:待拍板项被静默选边、范围靠推断、「建议未被反对」当「已同意」——错了不叫幻觉,叫「替用户做了他没做的决定」。封堵工具完全不同:追问/多变体/勾选/回显,按「偏好方差×返工成本」2×2 选用。全套见 [elicitation-protocol.md](elicitation-protocol.md);已烧进 [kit/explore-prompt-template.md](kit/explore-prompt-template.md)。

## 附:第六型——取证幻觉(2026-07-14 Euan gitleaks 事故增补)

完成幻觉的姊妹型:完成幻觉 = 谎称**做了**;取证幻觉 = 把没看见的细节补成**看见了**——工具输出里没有的 File/Line/Commit,用生态惯例与 plausibility 重建成「事实」报告。前 12 条堵的是「做」的缺口,这三条堵「查」的缺口(机制与模型档位无关,防线必须机械化,不赌模型自觉):

1. **细节缺失的标准动作 = 复跑取证**:本地复跑同版本工具、开详细模式(`-v`),绝不从记忆/生态惯例重建细节。
2. **「看过片段」≠「读过日志」**:grep/head 过滤后的视图只是片段,「没匹配到」≠「不存在」;基于片段下结论前先确认过滤器本身没坏(Euan 实证:日志行首 job 名前缀 `secrets` 让 grep 命中率 100%,过滤器失效未被察觉,「日志里没有细节」这个事实被淹没)。
3. **锚点出口前存在性检查**:报告 commit/路径/行号之前先跑 `git cat-file -e` / `ls`——让编造在出口前碎,不是在用户面前碎。

(Euan 全链条实证:CI 调 gitleaks 无 `-v` 只打统计行 → agent 编出「api/.dev.vars.example L7 · generic-api-key · commit 4a91c3…」全套似真细节 → `git log --all` 证实该文件全历史不存在、commit 号不存在;正确动作事后验证有效:本地开 `-v` 复跑一分钟拿到真 finding。)

## 附:开发期地板 vs 生产 API 门禁(2026-07-18 IPONews 实证)

**dev floor**(IDE SessionStart / PreToolUse / 本仓 preflight):防 **coding agent** 未读规则就 push、批跑、毁数据——装在开发者机器上。  
**prod API gate**:产品形态是「LLM → 你的 HTTP/API」(Vercel/Cloudflare Worker 等**无** Claude Code/Grok 运行时)时,防幻觉进金融/业务结论必须落在**程序层**(无 Evidence → refused、结构化字段只读 extractions、禁止裸 chat 当事实源)。

**禁止**把 hook 当生产护栏;两层互补、不互相替代。kit/claude-hooks-template 文首已钉「hook 非生产」。

## 一句话版

> **让 AI 少幻觉的方法,不是叮嘱它「别幻觉」,而是把它每一步的对错都变成可机械判定的事实:接口=同一份文件,状态=单写者台账,完成=命令输出,边界=禁令条款。缺口填完,幻觉自然没了生长的地方。**
