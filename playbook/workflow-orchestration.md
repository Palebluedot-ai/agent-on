# 工作流编排:确定性扇出的防幻觉七件

> 职责边界:本篇管「一次派出一群子代理(读 / 分析 / 各写各的文件)时,怎么让任务确定性跑完、且幻觉无法存活」;多 agent 改同一代码库的并行走六步协议(kit/AGENTS-skeleton.md §10),分工见 §〇。
> 源流:agent-on 自身融合工程实战(2026-07-07~08:三镜头设计评审 + 17 案例扇出 + 限流断点续跑,单日 22 代理),批四后消化落盘。原则工具无关——Claude Code 的 Workflow 工具是当前最顺手的载体,Codex 语境下见 §三。

## 〇、先分清用哪个:会不会撞文件?

- **会撞**(子任务改同一代码库、彼此有接口)→ **六步协议**:契约先冻、轨道 = worktree 物理隔离、契约测试当裁判
- **不会撞**(读 / 分析 / 各写各的文件,互不相交)→ **本篇工作流扇出**:不需要 worktree,需要下面七件

**大扇出前,探针先行(闸门)**:先派 1 个探针子代理,把本次扇出依赖的关键工具(调研类 = WebSearch 与 WebFetch)各实跑一次;探针不过,改主会话定向抓取,**不启动 fan-out**——环境性故障对每个子代理都成立,扇出只是把同一次失败乘以 N。(IPONews 同机正反对照:无探针 22 代理/18 错/66 万 token 零产出;有探针 105 代理/0 错。案例 24)

## 一、七件(每件一句为什么 + 本仓实战证据)

1. **控制流写成脚本,不靠模型记忆**。扇出几个代理、哪里设 barrier、谁先谁后,由确定性脚本决定——「忘了做某步」在结构上不可能;barrier 只设在真需要全量结果的点(评审必须等三案齐),其余全并行。
2. **返回必须过 schema**。子代理的交付强制为通过 JSON Schema 校验的结构化对象,不合格自动重试——想发散都没有语法空间。(实战:三提案 / 对抗评审 / 17 案例回执全部 schema 强制,零格式漂移。)
3. **任务书自包含 + 禁区显性**。源料精确路径、评估尺子、风格契约、返回格式、不许碰什么,一次给全;「只写点名文件、不跑 git」写进任务书——负空间不写,代理就替你决定。
4. **证据强制进任务书**。涉及可执行物,派工时就要求实跑并贴原始输出——「应该没问题」在任务书里判死刑。(实战:schema 轨贴了 4 个 JSON 校验 OK + 两次 lint 的 exit code。)
5. **对抗验证做成独立 stage**。产出后必接「假定有错,找到它」的审查代理,审查者与产出者绝不同一实例。(实战拦截:代际标错 9 处、audit-lint null bug、5 卡虚指一个不存在的清单、结账执行书对 S 档空转——没有一个是「大概没问题」混过去的。)
6. **失败语义显性 + 断点续跑**。代理死掉(限流 / 超时)返回空值、被过滤,不产生「我记得做过了」;重跑时已完成的走日志缓存、失败的重来;要强制重审的,改一行审查 prompt 击穿缓存。(实战:11 个案例代理被限流打死,resume 后成功者走缓存、失败者重跑、审查 prompt 加一行触发全量复审。)
7. **单一合流权**。代理只交付、不落地;orchestrator 逐件审查后分层 commit,每个 commit 可独立回滚——幻觉即使活过前六件,也进不了主干。

## 二、混编旋钮(与 model-playbook 联动)

- 执行型代理(读析写单件)配低档 effort / 便宜模型;**评审席升档**——对抗的持续性值钱
- 判断(合成、裁决、合流)永远留在 orchestrator 自己脑子里,不下放给任何子代理

## 三、穷尽式提取模式(「每个 X 都要留存/核对」类任务)

分片并行提取 → 合并 → **双路对抗核对** → 定稿。核对是命门:核对者必须**独立生成机械对表清单**(如从源码 grep 出全量状态名)再逐项对表,不许复读提取结果——复读式核对连提取者的漏项一起继承。(Euan 实证:9 代理提取 558 行状态清单,双路核对抓出 7 处勘误,含「渲染层覆盖死代码」「有逻辑无 UI 的隐藏能力」这类单路必漏项。)

## 四、边界与诚实

- 七件防的是「幻觉存活」,不是「幻觉产生」——模型仍会编,但编出来的过不了 schema、贴不出证据、活不过对抗审查、进不了主干
- Workflow 工具是 Claude Code 的能力;Codex 语境下第 1、6 件退化为手工纪律(编排步骤写成 checklist 逐项打勾、失败手工登记后重派),其余五件原样成立
- **扇出写作有代价**:需要统一作者声音的长文(README、宪章、合成裁决)不要扇出——素材可以扇出去收,成稿必须单脑
- 与防幻觉十二机制(anti-hallucination.md)的关系:十二机制是项目开发全程的通用封堵,本篇是「多代理扇出」这个特定场景的专用装备,同一个公式(幻觉 = 缺口 × 自由度)的两次应用
