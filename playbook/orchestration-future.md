# 编排能力的未来化:当模型从 Fable 换成 5.0、6.0,这套东西还剩什么?

> 2026-07-04,回答 Chao:「未来不一定是 Fable 或 Opus——这套编排怎么不被下一代模型淘汰?」
> 核心命题:**脚手架是保险,保费该随模型可靠性浮动;但有三样东西不是保险,是资产——模型越强它们越值钱。**

## 一、先分清:脚手架里什么是「保费」,什么是「资产」

### 三个不变量(资产,模型越强越值钱)

1. **契约即文件**(contracts/fixtures):它不是给弱模型的拐杖,是**多智能体的物理接口**。两个再强的模型并行,也需要一个双方都咬合的事实源——就像两个天才工程师也需要 API 文档。模型变强,契约不消失,只是**冻结成本趋近于零**(强模型一次写对)。
2. **单一状态写者**(progress.yaml/orchestrator):并发写状态的竞态是数学问题不是能力问题。GPT-9 也不能两个实例同时改同一个文件不出事。
3. **验证证据链**(测试输出/LIVE 实证/审查留痕):「完成=有证据」永远成立,因为它防的不是弱模型,是**任何智能体(包括人)的自利偏差**——执行者自评永远系统性偏乐观。S2 审查抓 Critical、basil 静默 null,都是证据链而非能力抓出来的。

### 可伸缩变量(保费,随模型可靠性调低)

| 旋钮 | 现在(Fable 5) | 模型更强时 |
|---|---|---|
| phase 卡颗粒度 | 半天一卡 | 一天一卡 → 一片一卡 |
| TDD 严格度 | 全量铁律 | 按爆炸半径分级:钱/数据全量,UI 抽查 |
| 审查轮次 | 每切片独立审查 | 只审高危切片,低危抽检 |
| 双轨 vs 单轨 | 前后端必拆 | 强模型可单 agent 全栈,并行只为提速不为隔离 |
| 悬点裁决 | 主会话逐条 | 双方模型自协商,主会话只裁分歧 |

**调法**:每换一代模型,跑一次「能力探针」——同一个标准小切片(比如一个 CRUD+攻击者测试),看它裸奔(低脚手架)的一次通过率。通过率 >95% 的环节,降一档保费。**用数据调,不用感觉调。**

## 二、为什么这套东西天然抗模型换代

本项目全部关键资产是**文件,不是 prompt**:

- 契约=JSON 文件,任何模型都能读
- 状态=YAML,任何模型都能读
- 知识=markdown(AGENTS.md/phase 卡/loop-notes),任何模型都能读
- 验收=测试代码,任何模型都能跑

**Prompt 是易挥发的,文件系统是持久的。**「自包含文件系统」当初为了防上下文丢失,副作用是天然的模型无关性——换模型=换执行引擎,资产层原地不动。Run #5 的断连续跑实证过:执行者失忆不要紧,世界状态在盘上。

## 三、编排本身的进化方向(给 agent-on 产品的三个押注)

1. **从「流程编排」到「风险编排」**:现在按工序拆(前端轨/后端轨),未来按**爆炸半径**拆——钱和数据的改动走重车道,其余走轻车道(freedom-vs-discipline.md 的二车道是第一步)。编排器的核心竞争力=**给每个任务自动定风险等级并匹配保费**,而不是给所有任务套同一套流程。
2. **评测集是护城河**:每个项目沉淀的「能力探针」+ 真实翻车案例(basil/微秒窗/适配器挂死)= 私有评测集。新模型来了,跑一遍就知道哪些保费能降。**模型会过时,评测集只会增值。**这是 agent-on 最该做成产品的部分。
3. **人的角色收敛到三件事**:品味注入(参照物/选原型)、闸门决策(D 表)、验收抽查。其余一切——包括编排本身——都应该是模型对模型。今天主会话当 orchestrator,明天 orchestrator 也是被编排的对象;但上面三件事收敛后依然在人手里,因为它们是偏好不是能力。

## 四、防幻觉与放能力的统一公式

「不想限死模型,又不想它幻觉」不是权衡,是分域:

> **幻觉率 = f(信息缺口 × 自由度)。**
> 在事实域(接口/状态/完成/决策):把信息缺口填死(12 机制),自由度随便给——没有缺口,自由不产生幻觉。
> 在创造域(视觉/文案/概念):信息缺口就是创作空间,自由度全开——那里的「幻觉」叫方案。
> 真正的错误只有一种:**把创造域的手法用在事实域**(让模型「发挥」接口形状),或反之(给视觉稿写单元测试)。
