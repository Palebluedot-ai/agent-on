# 工作流扇出 checklist(orchestrator 派工前照抄)

> 何时用:子任务是读 / 分析 / 各写各的文件、目标互不相交时。会撞文件就不用本单,改走 AGENTS 骨架 §10 六步协议。原理见 ../playbook/workflow-orchestration.md。
> 源流:agent-on 融合工程实战沉淀,2026-07-08 消化落盘。

## 派工任务书七要素(每个子代理的 prompt 必须齐)

- [ ] 1. **背景一段**:目标仓是什么、这个代理在整件事里的位置
- [ ] 2. **源料精确路径**:读哪几个文件,不让它猜、不让它翻
- [ ] 3. **评估的尺子**:判断标准、对齐哪份权威文件
- [ ] 4. **风格契约**:语言 / 格式 / 长度上限 / 文件头职责边界行
- [ ] 5. **返回 schema**:结构化字段定义,不许自由文本交差
- [ ] 6. **禁区**:只写点名文件、不跑 git、不碰哪些目录——负空间写全
- [ ] 7. **证据要求**:涉及可执行物必须实跑并贴原始输出

## 编排七件(脚本 / 流程侧)

- [ ] 大扇出前探针:1 个探针子代理先实跑本次依赖的关键工具(调研类 = WebSearch/WebFetch);不过 → 主会话定向抓取,不 fan-out(案例 24)
- [ ] 目标文件集互不相交(相交 → 六步协议)
- [ ] barrier 只设在真需要全量结果的点,其余全并行
- [ ] 对抗验证独立 stage:审查者 ≠ 产出者,prompt 带「假定有错,找到它」
- [ ] 失败 = 空值过滤,不默认成功;记下哪些代理死了、为什么
- [ ] 断点续跑方案先想好:什么走缓存 / 什么重跑 / 怎么击穿缓存强制重审
- [ ] 撞配额预案:已完成成果照用;缺口降档模型(sonnet/haiku)子代理续跑;合成判断留主会话(单会话实证)
- [ ] 合流单写者:逐件审查 → 分层 commit → 每个可独立回滚
- [ ] 完工触发扫尾:本次编排踩的坑当场进 loop-notes(六触发之一)

## 调研型扇出附加条款(价格 / 榜单 / 供应商阵容类)

- [ ] 直核证据 = 浏览器渲染 DOM,禁 WebFetch 摘要/纯文本;表格核对总行数防视区截断
- [ ] 型号名/价格当日官方页直核,逐项标查证日期;prompt 预填型号只当检索线索不当事实
