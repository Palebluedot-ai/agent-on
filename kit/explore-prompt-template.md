# Explore 车道派工模板(创造域专用)

> 与 track-prompt-template(Ship 用)配对。Explore 的风险不是事实幻觉,是**偏好缺口被静默填充**(elicitation-protocol.md)。本模板把五条协议烧死在结构里。

## 派工前:orchestrator 自查三件(30 秒)

- [ ] **待拍板对齐**:翻 brief/台账,凡「待拍板」维度→在 prompt 里显式标注「此维度做成可切换/变体,禁止选边」
- [ ] **范围勾选已发**:页面/功能清单类范围,先让用户勾过;没勾过就把范围本身做成变体
- [ ] **参照物已回显**:用户给的每个截图/链接,已用一句话「拿它的 X 用在 Y」确认过

## Prompt 骨架

```
你是【🏎 Explore 车道】[任务名]。只许写 [唯一产出路径];禁碰 [其余目录清单]。
先读:[brief/评审记录路径]、[token 路径]。

## 变体要求(协议 2:首稿必多变体)
产出 [N≥2] 个真正不同的 [方向/主题/方案],[并列展示/运行时切换] 供用户投票;
每个变体一句话设计意图。**用户已拍板的维度**([列出])照拍板执行,不再发散。

## 待拍板维度(协议 1:禁止选边)
[逐条列台账里的待定项]→ 每条做成 [切换器/并列变体],UI 上可比较。

## 三条底线(Explore 也要守)
1. 色值/字号只从 [design tokens 路径] 取,可组合不可发明
2. 真实感假数据用 [用户真实工作流样例]
3. [平台底线:响应式/reduced-motion/内容不锁 JS]

## 完成报告
- 产物路径;各变体一句话卖点;你的一票+理由(用户参考不受限)
- 需要用户拍板的新发现(≤4 个)
- 不 commit(处置权归主会话)
```

## 记住那组对照数据

Calendar 画廊 4 方向 → 零返工;官网 v1 单方向 → 全返工。**变体的边际成本是线性的,返工的成本是全量的。**
