# 案例 14:「echo n 不是护栏」:CLI 在 agent 环境自动确认

> 层级:L2 教训 | 来源:Euan 实战 2026-07-04 | 入册:2026-07-07 批三

## 症状
用 `echo n | supabase config push` 想只看 diff 不应用,结果配置被真推上去:邮箱验证被关、MFA TOTP 被关、storage.vector 被开。喂进去的 `n` 完全没拦住。

## 根因
CLI 检测到非交互环境(--agent auto)直接自动确认,管道里的 `n` 被无视。更深一层:最小 config.toml 里未声明的键,被按「未声明=重置为 CLI 默认值」处理,而不是「未声明=不动」——安全闸门是被默认值悄悄关掉的。

## 修法
靠行为探测发现(注册后立即登录,验证已失效),10 分钟内声明式修复:把在乎的键全量写进 config.toml 再重推,复测闸门回位。

## 可复用规则
任何会写状态的 CLI,在 agent 环境下一律假定它自动 yes——喂 `n` 不算护栏,查 --dry-run/--agent 旗标才算;没有 dry-run 就先在无害目标上试行为。
