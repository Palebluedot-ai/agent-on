# 案例 23:sed 字符类事故——结构化文件禁用 shell 字面量替换

> 层级:L2 教训 | 来源:Euan 实战 2026-07-12(dashboard.html 整页炸毁,git checkout 恢复) | 入册:2026-07-12 第四次消化

## 症状
用 `sed 's|[一段中文长字面量]|新文案|'` 改 dashboard.html,结果页面每一处出现**其中任意单个字符**的地方都被替换成了整句——整页面目全非。

## 根因
sed 的 `[...]` 是**字符类**语法:方括号里的长字面量被解释成"匹配其中任一字符",而不是整串。中文长串里几十个字符全部命中。反直觉处:对人眼它是"一段要替换的话",对 sed 它是一张字符集。

## 修法
git checkout 恢复文件,改用编辑工具(精确锚点的 Edit)重做替换。

## 可复用规则
改结构化文件(HTML/JSON/YAML/代码)一律用编辑工具精确锚点替换;shell 流式替换(sed/awk)只许用于自己刚生成、确认无正则特殊字符的内容。

## 已固化到哪
本案例即固化落点;与案 14(echo n 不是护栏)同族——shell 工具在 agent 手里的语义陷阱,使用时机表并列。
