## 1. 项目愿景

`project-kickoff-os` 的目标，是成为一个 AI-Native 项目启动操作系统。

它要解决的不是“怎么写更多文档”，而是：

- 怎么把项目讲清楚
- 怎么识别哪些已经确认、哪些还没确认
- 怎么让文档尽量自包含
- 怎么让 agent 以后进入新项目时不用从零猜

## 2. 它解决的核心问题

很多项目不是死在实现，而是死在启动阶段：

- 需求没澄清
- 文档不收敛
- milestone 不清楚
- 规则没固化
- 不同 agent 用不同套路

这个项目的意义，就是把启动阶段方法论变成可复用资产。

## 3. 当前定位

这是一个 side project，但目标是服务未来多个业务项目。

它不绑定：

- 某一个公司
- 某一个技术栈
- 某一个模型
- 某一个插件

## 4. 第一版成功标准

第一版成功，不看模板多少，而看：

- 第二个新项目能不能更快启动
- 文档是不是更少漂移
- 模型是不是更少幻觉
- 沟通是不是更少来回反复
